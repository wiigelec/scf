# Planning

This directory contains durable SCF planning and coordination artifacts.

Planning artifacts define proposed scope, dependencies, sequencing, lifecycle
state, and acceptance criteria. They do not independently create or modify
normative repository authority. Accepted authority and the development process
governing each planned change remain controlling.

## Development-foundation planning

`BOOTSTRAP-TO-DEVELOPMENT-ROADMAP.md` records the non-normative roadmap for
moving the repository from bootstrap initialization to normal governed
development. It directly references the implementation issues, records their
dependency structure and planning status, and defines bootstrap-completion and
normal-development entry criteria.

## Product-design planning

No product implementation plan currently exists.

The first governed product-design artifact will be a high-level design scope
for the **Session Continuity Framework (SCF)**. It must be derived from the
accepted SCF authority hierarchy and may begin only when the applicable
bootstrap-to-development entry criteria have been satisfied.

Detailed schemas, runtime architecture, and implementation work must not be
introduced before their governing authority and bounded planning scope exist.
